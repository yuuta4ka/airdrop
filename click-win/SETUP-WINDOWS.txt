AIRDROP ON WINDOWS - HOW TO GET FILES WITHOUT BREAKING THEM
==========================================================

PROBLEM: ZIP / Telegram / USB can break .bat files (encoding, line endings).
         You may see errors like: 'ho' is not recognized, 'agnose.bat' ...

BEST WAY: git clone (fast, correct files)

  1. Install Git: https://git-scm.com/download/win
  2. Install Node.js: https://nodejs.org (then REBOOT PC)
  3. Open cmd and run:

     cd %USERPROFILE%\Desktop
     git clone https://github.com/yuuta4ka/airdrop.git
     cd airdrop
     click-win\Diagnose.bat
     click-win\Start.bat

  4. Open http://localhost:8080

UPDATE PROJECT (after changes on Mac):

     cd path\to\airdrop
     click-win\Update-Project.bat
     click-win\Start.bat

  Update-Project runs git pull + npm install automatically.
  Start.bat also installs missing packages (e.g. after new features).

  Manual update:
     git pull
     click-win\Fix-Dependencies.bat
     click-win\Start.bat

TROUBLESHOOT npm install:

  EPERM (operation not permitted):
     - Run Stop.bat, close all cmd windows
     - Run Fix-Dependencies.bat (installs only adm-zip + pdf-parse)
     - Pause antivirus on the airdrop folder if needed

  ECONNRESET (network):
     - Check internet or try phone hotspot
     - Run Fix-Dependencies.bat again (auto-retries 3 times)
     - Or in cmd: npm install adm-zip@0.5.18 pdf-parse@2.4.5

IF YOU MUST USE USB:
  - Copy the whole airdrop folder (not only click-win)
  - Do NOT re-zip on Mac with special encodings
  - After copy, run Diagnose.bat first

DO NOT:
  - Send .bat files through Telegram (they get corrupted)
  - Open site as file:///.../index.html (need Start.bat running)
